# Настройка DHCP‑сервера: опция `-dd` для циклического распределения IP

## Описание

Опция `-dd` активирует режим **циклического повторного использования IP‑адресов** из заданного пула. Она решает проблему нехватки адресов при частой смене MAC‑адресов на одном порту или у одного пользователя.

## Принцип работы

### Сценарий без `-dd` (стандартное поведение)

1. Сервер выделяет IP из пула **строго по порядку**.
2. Каждый новый MAC‑адрес получает следующий свободный IP.
3. При исчерпании пула сервер **отказывает** в выдаче адреса.

### Сценарий с `-dd` (циклическое распределение)

1. Сервер выделяет IP по порядку до исчерпания пула.
2. При следующем запросе (новый MAC) сервер **возвращается к началу пула** и выдаёт первый IP.
3. Процесс повторяется циклически.

## Практический пример

**Конфигурация:**
* Пул адресов: `192.168.1.10–192.168.1.12` (3 адреса).
* Включена опция `-dd`.

**Последовательность подключений:**

| № | MAC‑адрес | Выданный IP | Комментарий |
|---|----------|-----------|-----------|
| 1 | AA:BB:CC:00:01 | `192.168.1.10` | Первый адрес из пула |
| 2 | AA:BB:CC:00:02 | `192.168.1.11` | Второй адрес |
| 3 | AA:BB:CC:00:03 | `192.168.1.12` | Третий адрес, пул исчерпан |
| 4 | AA:BB:CC:00:04 | `192.168.1.10` | Цикл: возврат к первому адресу |
| 5 | AA:BB:CC:00:05 | `192.168.1.11` | Следующий в цикле |
